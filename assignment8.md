# 应用系统体系架构 --- 作业8（2025-11-20）

## 1. 作业要求

请你在大二开发的 **E-Book 系统** 的基础上，完成下列任务：

### A. MongoDB 与 MySQL 融合存储改造

将你认为合适的内容改造为在 **MongoDB** 中存储，例如：\
- 整张 Book 表\
- Book 中的封面图片\
- 内容介绍\
- 书评（Reviews）

你可以参考课程样例将数据分别存储在 **MySQL** 和 **MongoDB**
中，也可以将所有数据都存储在 MongoDB 中。\
若采用后者，需要确保系统功能 **完全正常实现**，包括：\
- 书籍浏览\
- 查询\
- 下订单\
- 管理库存

------------------------------------------------------------------------

### B. 基于 Neo4J 的图书标签图构建与搜索功能

1.  为每一本图书添加若干 **标签（Tags）**，表示其类型分类。\
2.  在 **Neo4J**
    中将这些标签构建成一张连通图（或树），用于表示标签的层次关系。
    -   例如：`Fiction` → `Scientific Fiction`
        之间用一条边连接，表示后者是前者的子分类。
3.  在系统中新增基于标签的搜索功能：
    -   用户选择某个标签\
    -   从 Neo4J 中找出：
        -   此标签本身\
        -   能通过 **两条边以内** 关联到的所有标签\
    -   在 **MySQL** 中根据这些标签筛选所有含有任意一个相关标签的图书\
    -   将搜索结果呈现给用户
